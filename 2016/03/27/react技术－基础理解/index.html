react技术－基础理解

发表于 2016-03-27   |     |  阅读次数

在网上看了阮一峰关于react的入门实例讲解，发现讲解深入浅出，很适合刚入门的童鞋，以下是我的一些学习感悟：

demo1-12，13是nodejs环境做的，我才疏学浅，尚未参透，但时候会再看看：

温馨提示：由于demo中引入了react-dom.js 和 JSXTransformer.js，所以写法较简单易懂；

demo1:入门级，hello world一样的东西

主要收获：

  • 多个render要分开写，画几个地方写几个render；

demo2: 数组遍历

主要收获：

  • 很常规的东西，做项目会用到很多；

demo3: 还是数组遍历，但是数组里面多了html标签

主要收获：

  • 直接把数组画出来了，并没有什么新意

demo4: 组件的使用，还有就是外部变量props传入render使用

主要收获：

  • 发现个问题，如果接连使用的react，前一个出问题了，后一个会跟着遭殃，没法看

demo5: 数组遍历和prop赋值的双重天

主要收获：

  • 有些东西可以一起做。。。and 不推荐直接在body上画

demo6: 对props的类型有限定

主要收获：

  • React.PropTypes.string.isRequired写法和reactpropTypes.string.isRequired貌似是一样的，分别在这里的demo和facebook官网中见到过；

demo7 : 有除了render外其它的函数handleClick

主要收获：

  • render中的dom结构属性都可以this出来,见代码中的ref

demo 8: 点击切换，使用很广泛啊

主要收获：

  • 用到了三元还有一个新函数，render初始状态的设置getInitialState

###demo9: 内部传值的使用state

主要收获：

  • event.target.value指的是当前输入框中的值

demo10: 不透明度的渐变，扩展应用也是很广的哦

主要收获：

  • 新的函数componentDidMount，组件推入后的回调;

  • 还有在componentDidMount中的setInterval函数，是以整个进程的时间线进行的，它会间断持续进行；

  • 还有注意render中style的两次｛｝嵌套，据说是：一层是js的语法；一层是react的语法，记住哦，以后非属性就都这么写，不过貌似除了非属性就是属性嘛。。。。。

demo 11 : url获取值操作

主要收获：

  • $.get(url, function(){}.bind(this))这种写法好诡异，请求值后还要绑定到当前的组件上；

demo12: 终于最后了。。请求一个url，获取json数据，在render中对返回状态进行判断处理

主要收获：

  • 发现个问题，getInitialState都要加个return

  • 还用到了promise的then处理

  • getJson放到promise中，可以做返回数据是否成功的判断
